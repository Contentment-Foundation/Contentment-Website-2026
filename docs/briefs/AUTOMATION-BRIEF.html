Automation Principle & Architecture
Event Flow — All Integrations
Site form / Keela webhook
  └── Vercel Serverless Fn (POST /api/*)
        ├── Slack Webhook — team notifications
        ├── Flodesk API — segment + sequence
        ├── Zoom API — webinar registration
        ├── Google Sheets API — lead logging
        ├── Google Calendar — event creation
        └── Google Drive — story content

Third-party events
  └── Zapier / Make.com — no-code glue layer
        ├── Keela webhook → Sheets + Slack
        ├── Flodesk event → Sheets log
        └── Cron Monday 8am → Metrics digest

Decision rule: Write a Vercel fn when the trigger is a user-facing form POST and you need rate limiting, honeypot checks, or an immediate browser response. Use Zapier/Make.com when the trigger is a third-party system event (Keela webhook, Flodesk event, Sheets row) and no browser is waiting on the result.

donation.refundedSlack #errors alert

Keela Webhook handler — /api/keela-webhook

1
Verify HMAC-SHA256 signature using KEELA_WEBHOOK_SECRET — reject if invalid (prevents spoofing)
2
Parse event_type from payload body
3
recurring.created: POST Slack #homeroom → append Sheets row → Flodesk add to "homeroom-members"
4
recurring.cancelled: Slack cancellation alert → Flodesk move to "homeroom-churned"
5
Return 200 OK immediately — Keela retries on non-200 responses

Zapier

  • Best for simple 2-step zaps; non-technical team members
  • Free tier: 100 tasks/month · Starter: ~$20/month
  • Recommended if the ops team manages automations

Make.com

  • Best for complex multi-step flows; conditional logic; data transforms
  • Free tier: 1,000 operations/month · Core: $9/month
  • Recommended if an engineer sets up and owns the flows

Security Notes
✓All Slack webhook URLs and Keela HMAC secrets stored in Vercel env — never in code or Slack messages
✓Zoom credentials are Server-to-Server OAuth — no user login flow, no token in browser
✓Google Sheets API calls through service account (GCP_SERVICE_ACCOUNT_JSON) — server-only
✓Validate all inbound webhooks: HMAC for Keela; verify User-Agent and IP range for Zoom
✓Rate-limit /api/event-rsvp and /api/keela-webhook at Vercel edge (5 req / 15 min / IP)
✓Honeypot field on all site forms — checked server-side before any downstream calls fire
