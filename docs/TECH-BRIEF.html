Core Principle

Static-first. No custom server in Phase 1. Third-party services (Keela, Flodesk, Raisely) own the data. A Vercel backend + GCP Cloud SQL is added only when a provider cannot meet the workflow need.

Integration Specs

Flodesk — Newsletter

// Option B — custom form (full UI control)
POST /api/newsletter
Body: { "email", "first_name", "source" }

// Server (Vercel fn):
POST https://api.flodesk.com/v1/subscribers
Auth: Basic FLODESK_API_KEY ← server only
200 → inline success message
dup → "You're already subscribed"
5xx → friendly error + mailto

School Discovery Form

POST /api/school-inquiry
Body: {
"school_name" required
"contact_name" required
"contact_email" required
"role", "country", "message" optional
"_honey" ← honeypot, must be empty
}

// Server: Flodesk/Keela OR GCP INSERT
// Rate limit: 5 req / 15 min / IP
200 → thank-you (voice & tone)
5xx → friendly error + mailto

Homeroom Gate — Phase 2
// Vercel Edge Middleware (middleware.ts)
1. Request hits /homeroom/*
2. Check cookie: homeroom_session
3. Missing → redirect to /homeroom/login
4. POST /api/homeroom-auth
compare to HOMEROOM_GATE_PASSWORD (env)
rate limit: 5 attempts / 15 min / IP
success → HTTP-only cookie, 7-day TTL

// Phase 2.1: magic link (>500 members)
Vercel fn sends signed link → JWT cookie
Requires GCP homeroom_members table
GCP Cloud SQL Schema — Phase 2+ (skip at launch if Flodesk/Keela/Raisely suffice)
-- School leads (add only if you need queryable lead workflow)
CREATE TABLE school_inquiries (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  created_at TIMESTAMPTZ DEFAULT now(),
  school_name TEXT NOT NULL,
  contact_name TEXT NOT NULL,
  contact_email TEXT NOT NULL,
  role, country, message, source_page TEXT,
  status TEXT DEFAULT 'new' -- new | contacted | qualified | closed
);

-- Homeroom members (magic-link auth only)
CREATE TABLE homeroom_members (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email TEXT UNIQUE NOT NULL, -- matches Keela donor email
  keela_contact_id TEXT, tier TEXT, active BOOLEAN DEFAULT true,
  last_verified_at TIMESTAMPTZ
); -- All DB access via Vercel Functions only — never from browser
